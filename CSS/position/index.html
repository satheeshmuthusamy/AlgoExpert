Positions

  1. static==>The default value, the element follows the flow of the content

  2. fixed==>The element is positioned relative to the viewport and removed from the normal flow of the document. The top, right ,left and bottom properties can be used to move the element

  3. relative==> The element is positioned in the same place as it would be eith static . however it can be responded with top,left,right and bottom relative to its natural position

  4. sticky==> The element will be similar to relative positioned element ,but once it scrolls off it will stay to fixed to screen, essentially acting as position : fixed. This is particularly useful for menu bars that need to "stick" to the top if the screen as the user scrolls

  5. absolute==>BY default this will act as fixed except element will be relative to the document instead of viewport. This means that as the page is scrolled it will move with the page rather than stay at the same viewport location. However if any element above it in the DOM, oftentimes referred to as an ancestor, has a position value each other than static ,then it will be positioned to that nearest positioned ancestor.
